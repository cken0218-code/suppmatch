# Skill Scanner - 智能技能扫描器（优化版）

## 更新说明（v2.0）
- ✅ 白天轻量扫描（低 Token 消耗）
- ✅ 深夜深度分析（高 Token 消耗）
- ✅ 完整记录系统（不会忘记）
- ✅ 智能通知策略

## 执行策略

### 🌞 白天模式（08:00-23:00）
**目标**: 快速扫描，只记录基本信息

**执行内容**:
1. 扫描 ClawHub 最新 skills（limit 10）
2. VirusTotal 快速检查
3. 只记录基本信息：
   - Skill 名称
   - 描述
   - 安全状态（✅/⚠️/❌）
   - 分类
4. 不进行深度分析
5. **不通知用户**（除非发现高风险）

**记录格式** (`memory/skill-scans/YYYY-MM-DD-light.md`):
```markdown
## 12:00 轻量扫描
- skill-1 ✅ 安全 - 数据分析类
- skill-2 ⚠️ 可疑 - 自动化类
- skill-3 ✅ 安全 - 内容创作类
```

### 🌙 深夜模式（23:00-08:00）
**目标**: 深度分析，详细记录，概念提取

**执行内容**:
1. 详细审查白天发现的 skills
2. 代码分析（如果可以访问）
3. 提取核心概念
4. 设计安全替代方案
5. 生成详细报告
6. **整理所有记录**

**记录格式** (`memory/skill-scans/YYYY-MM-DD-deep.md`):
```markdown
## 02:00 深度分析

### 安全 Skills（可以安装）
#### skill-1
- **功能**: xxx
- **代码审查**: ✅ 无风险
- **使用场景**: xxx
- **安装命令**: `clawhub install skill-1`

### 概念学习（不安全但有用）
#### skill-2
- **风险**: VirusTotal 标记
- **核心概念**: xxx
- **可借鉴功能**: xxx
- **自行开发方向**: xxx

### 统计
- 今日扫描: X 个
- 安全: Y 个
- 可疑: Z 个
- 新增概念: W 个
```

## 记录系统

### 📝 必须记录的内容

**每个 skill 都要记录**:
1. ✅ 名称
2. ✅ 描述
3. ✅ 安全状态
4. ✅ 分类
5. ✅ 发现时间
6. ✅ 来源（ClawHub/GitHub）
7. ✅ 为什么安全/不安全

**绝不遗漏**:
- 即使是深夜扫描，也要完整记录
- 所有发现都要保存到文件
- 定期整理到 MEMORY.md

### 📂 文件结构

```
memory/
├── skills-safe-to-install.md       # 安全 skills 列表（随时更新）
├── skills-concepts-to-develop.md   # 概念记录（随时更新）
├── skill-scans/
│   ├── 2026-02-18-light.md        # 白天轻量扫描记录
│   ├── 2026-02-18-deep.md         # 深夜深度分析记录
│   └── 2026-02-18-summary.md      # 每日总结
```

## 通知策略

### 白天（不通知，除非）
- 🔴 发现高风险恶意 skill
- 🟢 发现用户急需的安全 skill
- 🟡 ClawHub rate limit 限制

### 深夜（不通知）
- 静默执行
- 只记录到文件

## 用户查询响应

当用户问：
- "有冇新 skills？" → 读取最新记录
- "上次发现咩？" → 显示 summary
- "帮我装 X skill" → 检查安全状态后执行
- "点解唔记得？" → **不可能忘记**，所有记录永久保存

## Token 优化

### 白天（节省 Token）
- 不执行深度分析
- 只记录基本信息
- 不生成详细报告
- 不通知用户

### 深夜（可以使用更多 Token）
- 完整代码审查
- 概念提取
- 详细报告生成
- 整理记录

## 安全检查清单

每个 skill 必须检查：
- [ ] VirusTotal 状态
- [ ] 作者可信度
- [ ] 代码可读性
- [ ] 权限请求
- [ ] 外部依赖
- [ ] 网络请求
- [ ] 敏感信息处理
