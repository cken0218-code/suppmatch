# AUTO-SAVE.md - 自動歸檔系統

## 🤖 運作邏輯

每次對話中，如果檢測到以下內容，**自動歸檔**到相應位置：

---

## 📂 分類規則

### 💰 生意資料庫 → `memory/business/`

**觸發詞：**
- 客戶、客、client
- 報價、quotation、quote
- Invoice、收據
- 生意、deal、合作
- 價錢、費用、收費
- 合約、contract

**歸檔位置：**
- 客戶資料 → `clients.md`
- 報價記錄 → `quotations.md`

---

### 📅 工作資料庫 → `memory/work/`

**觸發詞：**
- 開會、meeting
- 排程、schedule、時間表
- Deadline、截稿
- 會議、討論
- 跟進、follow up
- 任務、task

**歸檔位置：**
- 會議記錄 → `meetings.md`
- 排程事項 → `schedules.md`

---

### 💡 學習筆記 → `memory/learning/`

**觸發詞：**
- 靈感、idea、諗到
- 新嘗試、試下
- Code、function、API
- Bug、error、fix
- 技術、學習、教程

**歸檔位置：**
- 靈感想法 → `ideas.md`
- Code 相關 → `code-snippets.md`

---

## 🎯 執行時機

1. **即時歸檔**（高優先）
   - 客戶報價資訊
   - 會議安排
   - 重要靈感

2. **Heartbeat 整理**（定期）
   - 回顧近期對話
   - 補漏歸檔
   - 整理格式

---

## 📝 歸檔格式

每個檔案都有標準格式，確保一致性：

```
### [標題]
- 日期：YYYY-MM-DD
- [其他相關欄位]
- 備註：
```

---

## ⚠️ 注意事項

- 自動歸檔後會喺回覆中提及（簡短提示）
- 如果分類錯誤，話我知，我會幫你移動
- 敏感資料唔會外洩（所有檔案都係本地）
- 定期整理，避免檔案太亂

---

## 🔄 維護

- **每週**：檢視歸檔情況
- **每月**：整理格式 + 清理過時資料
- **按需**：調整分類規則
