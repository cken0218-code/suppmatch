- **作者:** ArulmozhiV

- **类型:** 数据爬取/浏览器自动化

**核心功能:**

1. **Profile Discovery** - 通过 Google Custom Search API 或 DuckDuckGo 发现 Twitter 账号
2. **Browser Scraping** - 使用 Playwright 进行浏览器爬取，配备反检测技术

**技术实现:**
- Playwright + Chromium 浏览器自动化
- 4 种轮换浏览器指纹配置
- 高级反检测 JavaScript（隐藏 navigator.webdriver、伪造 Canvas 指纹、模拟人类行为）
- 无需登录即可爬取公开资料

**🚨 安全风险评估:**

| 风险类别 | 等级 | 说明 |
|---------|------|------|
| **法律合规** | 🔴 高 | 可能违反 Twitter/X 的服务条款，存在法律风险 |
| **滥用潜力** | 🔴 高 | 大规模爬取可用于垃圾信息、虚假账户、社交工程攻击 |
| **反检测技术** | 🔴 高 | 专门设计用于绕过平台的安全措施 |
| **数据隐私** | 🟡 中 | 爬取用户公开数据，但可能被用于恶意目的 |

**详细风险分析:**
1. **反检测机制:** 该 skill 包含复杂的反检测技术，包括浏览器指纹伪造、行为模拟等，这些技术通常用于：
   - 规避平台的安全检测
   - 进行大规模的自动化攻击
   - 创建虚假账户

2. **配置灵活性:** 支持按位置、类别批量爬取，可配置最小粉丝数，最大推文数等，具有明显的规模化滥用潜力

3. **数据输出:** 详细的用户数据输出（用户名、简介、粉丝数、推文等），可被用于：
   - 钓鱼攻击的目标选择
   - 虚假信息传播
   - 社会工程攻击

**✅ 安全替代方案:**
- **官方 API:** 使用 Twitter/X 官方 API 获取数据
- **数据聚合服务:** 使用合法的数据提供商如 GNY 或 Datastream
- **手动研究:** 对于小规模需求，人工获取公开信息更安全
