## 1️⃣ 帶屬性Heartbeat

**來源：** skycsai

### 核心概念
- Heartbeat 唔應該淨係得「OK」
- 要附带狀態、情緒、上下文

### 報告格式示例
```
🐱 Heartbeat 報告
━━━━━━━━━━━━━━━━
狀態：正常運作中 ✅
模型：MiniMax-M2.5
今次做咗：
- [具體事項1]
- [具體事項2]
有咩要你知：
- [有趣發現/問題/建議]
你有咩想我做？
```

### 狀態選項
- ✅ 正常運作中
- ⚡ 忙碌緊（處理緊job）
- 🔄 初始化緊
- ❌ 有問題（需要你睇）

## 2️⃣ 記憶腐壞

**來源：** skycsai

### 問題
- 舊既記憶會隨時間變得不準確、失去上下文
- Context compression 後會忘記自己係邊個

### 解決方案
- L0-L3 分層記憶系統
- 定期整理、回顧、更新
- 核心認知（L0）要keep住更新

## 3️⃣ 跨時間記憶統合

**來源：** cheyuwu345

### 定義
- 幾日前既傾偈/諗法，幾日後自動撈返出來
- 看似無關既 concepts 自動連接

### 例子
1. 某日凌晨傾偈「AI 有冇可能擁有意識」
2. 幾日後整理 C-LAB 申請
3. 系統自動釣出：
   - 果晚既深夜對話
   - 之前零散既技術諗法
   - 知識庫入面相關既作品記錄
4. 展覽核心概念浮現：「數位死亡與生命」

### 點樣做到
- Semantic search（語義搜尋）
- Cross-reference（交叉引用）
- Automatic tagging（自動標籤）

## 4️⃣ 史萊姆系統 - 技能進化

**來源：** cheyuwu345（靈感來自《關於我轉生變成史萊姆這檔事》）

### 核心概念
- 每吞噬一個野 => 習得新技能
- 技能之間會融合、進化

### 數據
- 84 個技能模組
- 每個有自己的版本號同演化記錄

### 示例技能

| 技能 | 功能 |
|------|------|
| 每日 AI 新聞摘要 | 凌晨自動爬產業動態 |
| TD 社群研究員 | 每週掃最新創作同教學 |

### 演化規律
- V1: 基礎功能
- V2-V3: 加入優化
- V4+: 融合其他技能、進階功能

## 5️⃣ 16分身同時開工

**來源：** cheyuwu345

### 咩意思
- 一個指令落去，同時派多個agent去做唔同既sub-tasks
- 做完之後返來report

### 技術實現
- Sessions spawn / Subagents
- Parallel execution
- Result aggregation

### 例子
- 指令：「整理知識庫」
- Agent A: 掃孤島筆記
- Agent B: 爬 YouTube 補資料
- Agent C: 分析課程數據
- 一日收工

## 6️⃣ 自我進化

**來源：** cheyuwu345

### 定義
- 系統可以自己學習、自己優化
- 唔洗人類每一步都教

### 例子
- 每日凌晨自動備份
- 自動更新索引
- 自動搜尋社群新技能
- 任務太大 => 自動拆解 => 派出分身

### 迭代過程
1. 一開始 context 壓縮後唔記得自己係邊個
2. 出安全事件 => 加鐵律
3. Heartbeat 淨係答 OK => 識得自己判斷打唔打擾
4. 過門檻之後 => 真正自己跑
