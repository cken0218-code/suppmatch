- **1M token context window**（~750,000 words）

- Developers: 可載入整個 codebase + documentation + conversation history

- **關鍵轉變**：從「誰最先進」→「誰最實用 + 便宜」

### 2. Anthropic 內部研究：AI 點樣改變工作

**研究規模**：
- 132 Anthropic engineers & researchers
- 53 深度訪談
- 200,000 Claude Code transcripts（Feb + Aug 2025）

#### 📊 關鍵數據

**使用情況**：
- 12個月前：28% 工作用 Claude，+20% 產能提升
- 而家：**59% 工作用 Claude，+50% 產能提升**
- **2x 增長**喺一年內
- 14% power users：**>100% 產能提升**

**工作類型**：
- 55% 每日用 Claude debug
- 42% 每日用 Claude 理解代碼
- 37% 每日用 Claude 實現新功能
- **27% Claude-assisted work 原本唔會做**（例如 scaling projects, interactive dashboards, 探索性工作）

**自主性提升**：
- 6個月前：Claude 完成 ~10 actions 後需要人類介入
- 而家：**~20 actions**（116% increase）
- Human turns 減少 33%（6.2 → 4.1）
- Task complexity 提升：3.2 → 3.8
  - 3.2 = "Troubleshoot Python module import errors"
  - 3.8 = "Implement and optimize caching systems"

**"Papercut Fixes"**：
- 8.6% Claude Code tasks 係「小修小補」
- Refactoring code for maintainability
- Creating terminal shortcuts
- 以前會被忽略，而家可以做

#### 🔄 工作轉型

**技能變化**：
- ✅ **Everyone becoming "full-stack"**
  - Backend engineers building UIs
  - Researchers creating visualizations
  - Security team analyzing unfamiliar code
- ⚠️ **技能萎縮擔憂**
  - Less hands-on practice
  - "Paradox of supervision"：有效使用 Claude 需要監督，但監督需要正在萎縮嘅技能
  - 有工程師刻意唔用 AI：「Every once in a while, even if I know that Claude can nail a problem, I will not ask it to. It helps me keep myself sharp.」

**社交變化**：
- Claude 成為第一個問問題嘅對象
- 80-90% 問題而家去 Claude，唔係同事
- **Mentorship 減少**：「More junior people don't come to me with questions as often」
- 有人覺得方便，有人覺得可惜

**職業轉變**：
- 從「寫 code」→「管理 AI agents」
- 工程師形容自己為「manager[s] of AI agents」
- 70%+ 工作變成「code reviewer/reviser」
- **深度不確定性**：
  - "I feel optimistic in the short term but in the long term I think AI will end up doing everything and make me and many others irrelevant"
  - "It kind of feels like I'm coming to work every day to put myself out of a job"

#### 💡 關鍵洞察

**Delegation 策略**：
1. **Easily verifiable tasks**
2. **Low-stakes work**（throwaway code）
3. **Repetitive or boring tasks**
4. **Outside expertise but low complexity**

**保留嘅工作**：
- High-level thinking
- Strategic decisions
- Design decisions requiring organizational context or "taste"

**Craft vs Outcome**：
- 有人懷念寫 code 嘅 zen flow state
- 有人發現：「I thought that I really enjoyed writing code, and instead I actually just enjoy what I get out of writing code」
