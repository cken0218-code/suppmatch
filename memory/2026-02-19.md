## Multi-Agent Architecture Learning (From shikiouo's shared article)

### Key Learnings: OpenClaw Multi-Agent Operating System

**Core Philosophy**: Not "multiple bots chatting separately", but a real multi-role collaborative OS.

**5 Roles Architecture**:
1. **總指揮 (CEO)** - Global situational awareness, task breakdown, dispatch, correction
2. **軍師 (Strategist)** - Strategy analysis, plan evaluation, risk prediction
3. **工程師 (Engineer)** - Technical execution, code implementation, system maintenance
4. **創作官 (Creator)** - Content creation, expression optimization, external output
5. **智庫 (Knowledge Hub)** - Knowledge review, quality control, compliance check

**Architecture Principles**:
- Single Gateway + Multi-Agent + Multi-Workspace + Multi-Channel
- Each agent has independent workspace (personality, rules, memory, sessions isolated)
- Discord + Telegram dual-channel support
- Bindings-based precise message routing

**Key Insights**:
- "This is not hiring 5 people and throwing them in a room to freely interact. This is building a company - with organizational structure, job descriptions, communication protocols, private offices, meeting rules."
- "If you want multiple agents to cooperate in groups, just choose Discord. One platform is enough, others are not perfect."

**Correlation with autonomous-runner**:
- The "dispatch → execute → report" cycle aligns with autonomous runner concept
- Need to learn: How does routing work? How to track progress? Auto-notification?
